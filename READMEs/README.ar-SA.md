## ⚡ الخلاصة

**simplicio-tasks** هي **مهارة** واحدة مستقلّة عن بيئة التشغيل، تحوّل أي نموذج لغوي قوي
(Claude أو Codex أو Copilot أو Gemini أو Grok أو النماذج المحلية) إلى **منسّق ذاتي
الحركة يعمل بحلقة متكررة**. توجّهه نحو مجموعة من الأعمال — *"أنهِ كل القضايا المفتوحة"*،
*"أفرغ طابور الـ CI"*، *"صفّ لوحة Jira"* — وهو يدير دورة الحياة كاملةً بنفسه:

> **اكتشف ← افهم ← قرّر ← نفّذ ← تحقّق ← صحّح ← سجّل ← كرّر**

يكتشف الأعمال من أي مصدر، ويزيل التكرارات، ويوسّع تلقائياً أسطولاً من الوكلاء بما يناسب
جهازك، ثم ينفّذ كل عنصر عبر حلقة جودة **تُشغّل الشيفرة (لا تكتفي بتصريفها)**، ويفتح طلبات
الدمج، ويعالج ملاحظات الـ CI والمراجعة، ويدمج، ويواصل المراقبة **على مدار الساعة طوال
أيام الأسبوع** بحثاً عن أعمال جديدة — كل ذلك خلف بوابات أمان ومفتاح إيقاف صارم للتكلفة.

يحمل **43 نقطة توسعة مُسمّاة**. لكلٍّ منها بديل احتياطي يعمل دائماً عبر النموذج اللغوي،
وكلٌّ منها *يرتبط بأمرٍ أصيل في بيئة التشغيل المضيفة* عند توفّره — مما يجعل الخطوة حتميّة
وشبه عديمة الرموز. **المهارة لا تسمّي أي بيئة تشغيل؛ بل بيئة التشغيل هي التي تكتشف المهارة.**
هذا الانعكاس هو سرّ الحيلة كلها: بروتوكول عالمي واحد، مع حقن اختياري للسرعة الأصيلة من تحته.

</div>

```text
/simplicio-tasks termine as issues abertas
→ identity + pre-flight (kill-switch, auth, watcher)
→ discover 50 issues · dedup · build dependency DAG
→ autoscale fleet = 14 · pipeline implement→review→merge
→ each item: read body+ACs → orient code → plan → edit → run → verify → PR
→ merge · close with evidence · rollback if main breaks
→ keep polling every ~2 min for new work
```

## 🆚 مقارنة بـ caveman و rtk

بُنيت simplicio-tasks **بعد دراسة معمّقة** لأفضل أداتين لتوفير الرموز على GitHub —
[**caveman**](https://github.com/JuliusBrussee/caveman) (74 ألف ★، *ضغط الكلام*)
و[**rtk**](https://github.com/rtk-ai/rtk) (63 ألف ★، *ضغط الأوامر*). وهي تدمج أفضل ما في
**كليهما** ضمن منسّق متكامل. فهما يقلّلان الرموز؛ بينما simplicio-tasks **تنجز العمل**
وتقلّل الرموز أثناء إنجازه.

| | 🪨 **caveman** | ⚙️ **rtk** | 🔁 **simplicio-tasks** |
|---|---|---|---|
| **ما هي** | مهارة Claude Code | وسيط CLI بلغة Rust | مهارة مستقلّة عن بيئة التشغيل |
| **الفكرة الجوهرية** | كلام أقصر (حذف الحشو) | تقليل مُخرَجات أوامر التطوير | **تنسيق المهمة بأكملها** |
| **النطاق** | مُخرَجات النصوص من النموذج | مُخرَجات أوامر الصدفة (Shell) | دورة حياة العمل كاملةً من البداية للنهاية |
| **توفير الرموز** | ~65% في الردود | 60–90% في الأوامر | كلاهما — كتالوج + سقوف + تقليم |
| **هل تنجز العمل؟** | ❌ تنسيق فقط | ❌ وساطة فقط | ✅ اكتشاف←تنفيذ←دمج←إغلاق |
| **استقلالية متعددة الخطوات** | ❌ | ❌ | ✅ مجموعة عمّال مستمرّة |
| **بوابات الجودة** | — | — | ✅ بوابة معايير القبول · التحقق بالتشغيل · تحقق تخاصمي · بوابة التسليم |
| **الأمان** | — | semgrep، إخلاءات مسؤولية | ✅ حُكم رباعي الحالات · إثبات · فحص الأسرار · بوابة بشرية · مفتاح إيقاف |
| **الحلقة على مدار الساعة** | ❌ | ❌ | ✅ مراقب مُتين ذاتي الإصلاح |
| **الربط ببيئة التشغيل** | Claude/Codex/Gemini | أي بيئة (وسيط PATH) | **أي بيئة** (43 نقطة توسعة) |
| **ما أخذناه** | تقارير العمّال المقتضبة، طبقات الكثافة، حارس "لا تُعِد الصياغة أبداً"، أساس مرجعي صادق | كتالوج تقليل لكل أمر، سقوف متدرّجة بحسب الإشارة، تقليم مركّب، فشل آمن (fail-open)، حكم رباعي الحالات | — |
| **ما تركناه** | حذف كلمات القواعد (يضرّ جودة الشيفرة) | سجلّات خاصة بكل لغة (مرتبطة ببيئة بعينها) | — |

> لقد **رفضنا** عمداً أسلوب caveman في حذف الكلمات على طريقة "تكلّم كرجل الكهف" — فالنثر
> *المقتضب* لا بأس به، لكن تشويه القواعد يضرّ بالشيفرة والتأكيدات. احتفظنا بـ*الانضباط*
> (لا تُعِد صياغة الشيفرة أو الروابط أو المسارات أبداً)، لا بالحيلة.

## 🧩 نقاط التوسعة الـ 43

كل خطوة من العمل تحدث عند **نقطة توسعة مُسمّاة**. فإذا أتاحت بيئة التشغيل المضيفة قدرةً
أصيلةً فإنها **تربط** بها (حتميّة، شبه عديمة الرموز). وإلا قام النموذج اللغوي بتنفيذ
**البديل الاحتياطي** بأدوات معيارية (صدفة، git، gh، تحرير الملفات، الويب). المهارة تعتمد
على التجريد، لا على بيئة تشغيل بعينها أبداً.

| النقطة | ما تفعله |

## 📦 كل ما بالداخل

جردٌ كامل لما تحمله المهارة — كل آلية، موثّقة بمرجعها.

### الحلقة (7 خطوات + خطوات فرعية)
- **الخطوة 0** — تحميل العقد (البروتوكول القانوني).
- **الخطوة 1** — الهوية + كشف رخيص للبيئة.
- **الخطوة 1b** — نقاط التوسعة الـ 43 (ربط أصيل أو بديل احتياطي عبر النموذج اللغوي).
- **الخطوة 1c** — بوابة اقتصاد الرموز: `THINK / NO-THINK`، `INTERNET off by default`،
  `terminal-first execution`، **كتالوج تقليل المُخرَجات**، **سقوف متدرّجة بحسب الإشارة**،
  **طيّ النجاح + إزالة التكرار**، **تقليم الأوامر المركّبة**، **طبقات كثافة موجّهة بحسب
  المستهلك**، **فشل آمن (fail-open)**، **وضوح تلقائي (الأمان يَجُبّ الإيجاز)**.
- **الخطوة 1d** — ما قبل الإقلاع: رصيد مفتاح الإيقاف، مصادقة المصدر، تسليح المراقب.
- **الخطوة 2** — اكتشاف عناصر العمل وتوحيدها (أي موصّل مصدر).
- **الخطوة 2b** — استيعاب عميق: قراءة المتن والتعليقات كاملةً، استخراج **معايير القبول**،
  **توجيه قاعدة الشيفرة**، **وضع قراءة التوقيعات فقط**، وبناء خطة.
- **الخطوة 2c** — DAG التبعيات + الجدولة الطوبولوجية.
- **الخطوة 3** — موجِّه ثنائي المسار: مجموعة عمّال مستمرّة بـ **المسار السريع** مقابل
  **المسار الثقيل** · **عزل واعٍ بالتعارض** · **عقد تقرير العامل** · **ذاكرة تصحيحات**.
- **الخطوة 3b** — استيعاب مستمر: مستطلِع داخل التشغيل + مراقب خمول (يرصد العمل الجديد في
  أي دقيقة).
- **الخطوة 3c** — نموذج السرعة: خط أنابيب (لا حاجز)، مخزن تصريف مشترك،
  تحقُّق-مرة-واحدة-عند-الدمج، **خلاصة سياق مشتركة**.
- **الخطوة 3d** — توجيه النماذج L0←L4 (حتمي ← محلي ← متوسط ← استدلالي ← مدفوع).
- **الخطوة 4** — حلقة الجودة · **بوابة معايير القبول (تعريف إنجاز حقيقي)** · **التحقق
  بالتشغيل** · **تحقق تخاصمي متعدد الأصوات** · **بوابة التحليل الساكن**.
- **الخطوة 5** — بوابات الأمان: فحص الأسرار، بوابة بشرية للعمليات غير القابلة للتراجع،
  **حُكم رباعي الحالات قبل التنفيذ**، **إثبات مركّب لكل مقطع**، **ثقة قبل تحميل الإعداد**،
  **بوابة سلامة سلسلة التوريد**، **transform_guard**.
- **الخطوة 6** — التسليم + الإغلاق + التدقيق الذاتي · **حزمة الأدلة** · **التحقق من الواقع
  (لا تثق بالتقرير الذاتي أبداً)** · **حارس التراجع إذا كسر الدمج فرع main**.
- **الخطوة 6b** — إغلاق حلقة التغذية الراجعة: CI ← إصلاح، تعليقات المراجعة ← حلّ،
  تأخّر-الفرع ← مصالحة، **دورة حياة طلب الدمج** الكاملة حتى الجاهزية للدمج.
- **الخطوة 7** — حلقة قائمة على مدار الساعة (10 محاور): مُشغِّل مُتين، مصفوفة تغطية شاملة،
  حالة مُتينة، **حوكمة تكلفة + مفتاح إيقاف صارم**، أمان دون إشراف، إصلاح ذاتي + **إعادة
  محاولة ذكية بحسب صنف الفشل**، تحديد الأولويات/العمل قيد التنفيذ، رصد + **تدقيق دوري
  للتوفير** + **قياس باللقطات**، تحسين ذاتي، تنسيق وإيقاف نظيف.

### اقتصاد الرموز (مدمج من rtk + caveman)
- التنفيذ المُوجَّه نحو الطرفية أولاً — لا تحاكِ أمراً أبداً.
- جدول استبدال **متعدد المنصات** (Windows / macOS / Linux): أكثر من 30 حقيقة تجيب عنها
  الطرفية بكلفة أرخص من النموذج اللغوي.
- **كتالوج تقليل المُخرَجات** كبيانات: وصفة لكل أمر، نسبة التوفير المتوقعة %،
  حارس `skip-if-structured`.
- **سقوف متدرّجة بحسب الإشارة**: `CAP_ERRORS / CAP_WARNINGS / CAP_LIST / CAP_INVENTORY`.
- **طيّ النجاح** + **إزالة التكرار مع العدّ** (مع حارس `unless errors`).
- **تقليم الأوامر المركّبة** — لكل مقطع، آمن مع الأنابيب/إعادة التوجيه، فشل آمن.
- **طبقات كثافة بحسب المستهلك** (آلة مقابل إنسان)؛ تخطّي المحتوى الكثيف أصلاً.
- **عقد تقرير العامل** — مخطّط مقتضب يبدأ برمز الحالة للوكلاء الفرعيين.
- **أساس توفير صادق** = ذراع تحكّم واقعية، **مرتبطة ببوابة جودة ناجحة** (الضغط الذي يفشل
  في بوابته يحصل على صفر رصيد).

### الجودة والتسليم
- قائمة تحقّق تعريف الإنجاز بمعايير القبول · التحقق بالتشغيل · تحقق تخاصمي · بوابة التحليل
  الساكن · شهادة التسليم · إعادة التحقق من الواقع · تراجع تلقائي.

### الأمان
- فحص الأسرار · بوابة بشرية للعمليات غير القابلة للتراجع · حُكم رباعي الحالات (لا ترفع
  الصلاحيات أبداً) · إثبات الأوامر المركّبة · ثقة قبل التحميل · سلامة سلسلة التوريد ·
  تحصين ضد حقن التعليمات · مفتاح إيقاف صارم بالدولار للتشغيلات دون إشراف.

### الاستقلالية على مدار الساعة
- مجدوِل مُتين · طابور حيّ + مراقب خمول · سجلّ/حالة مُتينان · قواطع دائرية · حجر صحي
  للرسائل الميتة · تحسين ذاتي ومراجعة فوقية · مطالبات ذرّية متعددة النُسخ · إشارة إيقاف
  نظيفة (STOP).

## 🚀 التثبيت والاستخدام

simplicio-tasks هي **مهارة** — مجلّد واحد تُسقِطه داخل أي بيئة تشغيل تُحمّل المهارات. بلا
أي تبعيات، ودون الحاجة إلى ملف ثنائي.

</div>

```bash
# Claude Code (project or user skills dir)
git clone https://github.com/wesleysimplicio/simplicio-tasks
cp -r simplicio-tasks/.claude/skills/simplicio-tasks  <your-repo>/.claude/skills/

# then, in your agent:
/simplicio-tasks finish all the open issues
```

<div dir="rtl">

تُحمّل بيئات التشغيل الأخرى (Codex، Gemini، Copilot، الوكلاء المحليون) ملف
`SKILL.md` نفسه — راجع [`AGENTS.md`](../AGENTS.md) و[`CLAUDE.md`](../CLAUDE.md)
و[`GEMINI.md`](../GEMINI.md) لنقاط الدخول الخاصة بكل بيئة تشغيل. وحيثما تتيح بيئة التشغيل
المضيفة أوامر أصيلة، فإنها تربطها تلقائياً بنقاط التوسعة؛ وإلا فإن البدائل الاحتياطية
للنموذج اللغوي تغطّي **100%** من العمل.

**قبل تشغيل غير مُشرَف عليه على مدار الساعة:** اضبط سقفاً للتكلفة
(`.orchestrator/loop-budget.json`، `daily_usd_ceiling > 0`)، وتأكّد من أن مصادقة المصدر
دائمة، وأبقِ البوابة البشرية للعمليات غير القابلة للتراجع وفحص الأسرار مُفعَّلين. فإذا كان
`ceiling = 0` رفض المراقب التشغيل دون إشراف (فشل آمن).
